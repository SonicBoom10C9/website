---
title: Docker Compose
description: Run Sharkord using Docker Compose with Caddy as a reverse proxy.
icon: Container
---

This is probably the easiest way to run Sharkord using Docker. It uses Docker Compose to manage the containers and Caddy as a reverse proxy to handle HTTPS and domain routing.

To use this method, create a `docker-compose.yml` file with the following content:

```yaml
version: "3.9"

services:
  sharkord:
    image: sharkord/sharkord:latest
    container_name: sharkord
    restart: unless-stopped
    expose:
      - "4991" # internal only (no host binding)

    environment:
      SHARKORD_PORT: "4991"

    volumes:
      - ./data:/home/bun/.config/sharkord

  caddy:
    image: caddy:2-alpine
    container_name: sharkord-caddy
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"

    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile
      - caddy_data:/data
      - caddy_config:/config

    depends_on:
      - sharkord

  # WebRTC port must stay directly exposed on host
  # because media traffic does NOT go through reverse proxy
  sharkord-ports:
    image: alpine
    container_name: sharkord-ports
    network_mode: service:sharkord
    restart: unless-stopped
    ports:
      - "40000:40000/udp"
      - "40000:40000/tcp"
    command: ["sh", "-c", "sleep infinity"]

volumes:
  caddy_data:
  caddy_config:
```

Make sure to create a `Caddyfile` in the same directory with the following content, replacing `your-domain.com` with your actual domain name:

```
your-domain.com {
  reverse_proxy sharkord:4991
}
```

Then, run the following command in the terminal:

```bash
docker-compose up -d
```

This will start the Sharkord server and the Caddy reverse proxy. Caddy will automatically obtain and manage SSL certificates for your domain, so your Sharkord instance will be accessible securely via `https://your-domain.com`.
